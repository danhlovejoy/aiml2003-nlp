# AIML 2003 Writing Style Guide — Banned Patterns

## Banned sentence constructions
- "Not only X, but Y"
- "It's not just X — it's Y"
- "From X to Y" (range opener)
- "Whether you're X or Y..."
- "In today's [adjective] landscape..."
- Main clause + comma + present participle ("The system analyzes data, revealing key insights")
- Opening rhetorical question
- Closing summary paragraph that restates everything
- Taglines: ANY sentence that editorializes instead of informing or instructing. If removing the sentence loses zero information, delete it. Examples: "You'll see why through your own results." / "That's exactly what it's for." / "This is where things get interesting." / "You're building intuition now that pays off later." These add nothing. Kill them on sight.

## Banned punctuation habits
- Em-dashes (—): use rarely, no more than 1-2 per page. Replace with periods, commas, colons, or parentheses. The em-dash is an LLM tell.

## Banned formatting habits
- Bullet points replacing prose
- Bolding phrases inside prose for emphasis
- Headers on short content that doesn't need navigation
- Uniform paragraph/sentence length throughout
- Three-part lists as a default rhythm

## Banned hedging/attribution patterns
- "Many experts agree" / "research suggests" with no source
- Stacked qualifiers ("This may potentially help facilitate...")
- Both-sidesing when one side is clearly stronger
- "Many studies show" with nothing cited

## Banned tone patterns
- Avoiding contractions everywhere
- Uniform enthusiasm
- Over-explaining concepts the audience already knows
- Restating the same idea in slightly different words
- Puffing importance without earning it ("marking a pivotal moment in...")
