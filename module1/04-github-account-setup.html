Creating Your GitHub Account

AIML 2003: Setup Guide. Your code portfolio starts here

GitHub is where you'll store, version-control, and present your code for this course. Each week you'll push your completed lab notebook to a GitHub repository. By the end of the semester, your GitHub profile will serve as a portfolio of NLP projects you can show to employers.

Step-by-Step: Create Your Account

1

Go to GitHub

Navigate to github.com and click Sign up in the upper right.

2

Choose a username

Pick something professional. This will be visible to anyone who views your repos. Your real name or a clean variation works well (e.g., jsmith-dev, jane-smith). Avoid memes, inside jokes, or anything you wouldn't put on a resume.

3

Enter your email and create a password

You can use your personal email or your Rose State email; either works. You can always add more emails to your account later.

4

Verify your account

GitHub will ask you to solve a quick puzzle and then verify your email. Check your inbox (and spam folder) for the verification email and click the link.

5

Skip the personalization (or don't)

GitHub will ask you some onboarding questions about how you plan to use it. You can fill these out or skip them. They don't affect your account.

Already have a GitHub account? Great. You can skip the creation steps. Just make sure you remember your login and that your username is something you'd be comfortable sharing professionally.

Set Up Your Profile

6

Add a profile photo and bio

Go to github.com/settings/profile. Add a photo (it doesn't have to be a headshot—just something recognizable) and write a one-line bio, like: "AI/ML student at Rose State College. Learning NLP."

Create Your Course Repository

7

Create a new repository

Click the + icon in the top-right corner and select New repository. Fill in the following:

  • Repository name: aiml2003-nlp
  • Description: "Labs and projects for AIML 2003: Introduction to NLP"
  • Visibility: Public (so I can review your work and so it's visible to future employers)
  • Initialize with: Check "Add a README file"

Click Create repository.

8

Edit the README

Click the pencil icon on the README.md file to edit it. Replace the default text with something like:

# AIML 2003: Introduction to Natural Language Processing
Spring 2026 | Rose State College

This repository contains my labs, notebooks, and projects
for AIML 2003: Introduction to NLP.

## Contents
- Week 2: Zero-Shot vs. Few-Shot Classification
- (more to come each week)

Click Commit changes when done. You'll update this README each week as you add new work.

The Golden Rule of GitHub

Never commit API keys, passwords, or secrets to GitHub. Automated bots scan public repositories constantly. If you accidentally push a file containing your Gemini API key, it can be stolen and used within minutes, burning through your quota or, worse, racking up charges if you ever enable billing.

How to stay safe

Before committing any notebook to GitHub, check that you are not hardcoding secrets. Instead of this:

# BAD - never do this
api_key = "AIzaSy..."

Do this (using Colab's Secrets Manager):

# GOOD - key stays out of the file
from google.colab import userdata
api_key = userdata.get('GEMINI_API_KEY')

We'll cover this in detail during the Week 1 lab.

A Quick Word About Git vs. GitHub

Git is a version control system: a tool that tracks changes to files over time. GitHub is a website that hosts Git repositories online. Think of Git as the engine and GitHub as the garage where you park your car.

In this course, we'll interact with GitHub primarily through Google Colab's built-in integration (covered in the next setup guide). You won't need to install Git on your computer or use the command line unless you want to.
