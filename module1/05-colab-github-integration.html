Connecting Google Colab to GitHub

AIML 2003: Setup Guide. Save notebooks directly to your repo

Once Colab is connected to GitHub, you can save your work directly to your repository without downloading files or using the command line. This guide covers two methods: the built-in Colab menu (easiest) and a quick programmatic approach for when you want more control.

Method 1: Save via the Colab Menu (Recommended)

This is the simplest approach and the one we'll use for most labs.

1

Open a notebook in Colab

Open any notebook. You can use the week1_setup.ipynb you created in the Colab setup guide.

2

Go to File → Save a copy in GitHub

In the top menu, click File and then Save a copy in GitHub.

3

Authorize GitHub (first time only)

A GitHub authorization window will pop up. Click Authorize googlecolab. This grants Colab permission to write to your repositories. You only need to do this once.

4

Choose your repository and path

In the dialog that appears:

  • Repository: Select your-username/aiml2003-nlp
  • Branch: Leave as main
  • File path: Type a path like week1/week1_setup.ipynb to organize by week
  • Commit message: Write something descriptive, like "Add Week 1 setup notebook"
5

Click OK

Colab will save a copy of your notebook directly to your GitHub repository. You can verify by visiting your repo on github.com. You should see the new file.

Before you save: Double-check that your notebook does not contain any API keys, passwords, or other secrets. Look through all code cells for anything that looks like api_key = "sk-...". If you used Colab's Secrets Manager (from the Colab setup guide), you're safe.
—OR—

Method 2: Open a Notebook from GitHub

You can also go the other direction: open a notebook that's already in your GitHub repo directly in Colab.

1

Go to the Colab GitHub tab

Navigate to colab.research.google.com/github/. Sign in with your GitHub account if prompted.

2

Search for your repository

Type your GitHub username or select your repository from the dropdown. Colab will show all .ipynb files in that repo.

3

Click to open

Click on any notebook to open it in Colab. When you're done editing, use File → Save a copy in GitHub to push your changes back.

Pro tip: You can open any public GitHub notebook directly in Colab by swapping github.com for colab.research.google.com/github in the URL. Transform this URL: github.com/user/repo/blob/main/notebook.ipynb into this: colab.research.google.com/github/user/repo/blob/main/notebook.ipynb

Your Weekly Workflow

Here's the process you'll follow each week for labs:

  1. Start a new notebook in Colab (or open the starter notebook I provide).
  2. Work through the lab, running cells and adding your code and notes.
  3. Review your notebook before submitting. Make sure it runs top-to-bottom, your outputs are visible, and there are no exposed secrets.
  4. Save to GitHub using File → Save a copy in GitHub, placing it in the appropriate week folder (e.g., week2/lab1_classification.ipynb).
  5. Submit the GitHub link in Canvas.

Troubleshooting

I don't see my repo in the dropdown

Make sure you authorized Colab to access your GitHub account (Step 3 above). If you did but still don't see it, go to github.com/settings/applications, find "Google Colaboratory" in the list, and make sure it has access to your repositories.

The save failed with an error

This usually means the file path has an issue. Make sure folder names don't contain spaces or special characters. Use lowercase and hyphens (e.g., week2/lab1-classification.ipynb).

I accidentally committed a secret

Don't panic, but act fast. Immediately go to Google AI Studio and revoke/regenerate your key. Then remove the file from GitHub or edit the file to remove the key and commit again. Deleting the file is not enough. The important thing is to REVOKE the secret. Git history retains old versions. Ask me for help if you have trouble revoking your secret.
